# CLAUDE.md

## Project Rules Learned

### Rule 1: Write Clear Prompts
Always provide clear requirements, expected output, and project context. Detailed prompts produce more accurate and complete results than vague prompts.

### Rule 2: Include Constraints
Specify constraints such as responsive design, accessibility, required fields, validation, and output format. These constraints help the AI generate code that better matches project requirements.

### Rule 3: Verify AI-Generated Code
Do not accept AI-generated code without reviewing it. Check correctness, usability, accessibility, and whether it satisfies all project requirements before committing it.

### Rule 4: Use Version Control
Create separate Git branches for different prompt experiments. Commit each version with meaningful commit messages to make comparisons easier.

### Rule 5: Compare Before Finalizing
Use `git diff` to compare different AI-generated solutions and identify improvements before choosing the final implementation.
